What Is Base64?

Base64 is a binary-to-text encoding scheme that represents binary data in the form of a set of 64 printable ASCII characters. It is often necessary to use binary data in a system that only supports text. In this case, Base64 encoding can be used.

How It Works

For each set of 3 bytes of binary data, there are 4 Base64 characters. This means that the data is approximately 33% larger than the original but perfectly safe to send using any text-based protocol.

Base64 vs. Encryption

Base64 is encoding, not encryption. Anyone can decode it. It does not provide security. It's simply for safe transmission.

Is Base64 a form of encryption?

No, and this is a really common misconception. Base 64 is just encoding, and it does nothing to secure data. Anyone who looks at a Base 64 encoded string can immediately decode it.

Why does Base 64 output contain "==" sometimes?

Base 64 is a simple encoding scheme, and it works by taking 3 bytes of data and converting them into 4 characters of output. This means that if you're working with a data length that isn't a multiple of 3, you'll need to add padding characters to make it a multiple of 4. This is why you'll sometimes see "==" in a Base 64 encoded output, because this is a signal to a decoder that there are padding bytes in the data.

  • Working with HTTP Basic Authentication headers
  • Working with JWT token payloads
  • Sending email attachments
  • Working with binary data and a JSON API that only accepts text strings
