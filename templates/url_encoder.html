What Is URL Encoding?

URL encoding (also called percent-encoding) converts characters that are not allowed in a URL into a safe format. Each unsafe character is replaced by a percent sign (%) followed by two hexadecimal digits representing the character's ASCII code. For example, a space becomes %20 and an ampersand becomes %26.

This is essential when passing query parameters, form data, or special characters in URLs. Without proper encoding, browsers and servers may misinterpret the data.

encodeURIComponent vs encodeURI

  • encodeURIComponent, encodes everything except letters, digits, - _ . ! ~ * ' ( ). Use for query parameter values.
  • encodeURI, preserves URL structure characters like / ? # & =. Use for a complete URL.

When To Use URL Encoding

  • Passing search terms in query strings
  • Encoding form POST data
  • Building API request parameters
  • Sharing URLs with special characters
  • Debugging encoded URLs from server logs

Frequently Asked Questions

How do I encode a URL?

Paste your URL or text into the input box, make sure "Component" mode is selected for query parameter values (or "Full URL" if you're encoding a complete URL), then click "Encode". The percent-encoded result appears in the output. Hit "Copy" to use it in your code or browser.

What's the difference between Component and Full URL mode?

Component mode (encodeURIComponent) encodes nearly every special character, including / ? # & =, perfect for encoding a search query or parameter value. Full URL mode (encodeURI) leaves structural URL characters intact, so slashes and question marks stay as-is. Use Component for values, Full URL for complete addresses.

Why does a space become %20 in URLs?

URLs can only contain a limited set of ASCII characters. Any character outside that set, including spaces, gets "percent-encoded": replaced by a % sign and the character's two-digit hexadecimal code. A space is ASCII character 32, which is 20 in hex, hence %20. Some forms use + to represent spaces in query strings, but %20 is the correct standard.

When do I need to decode a URL?

Decoding is useful when you receive a URL from a server log, a redirect, or an API response and need to read what the encoded values actually say. For example, a URL like q=hello%20world%21 decodes to q=hello world!. Paste any encoded URL here and click "Decode" to make it human-readable again.

No. Everything happens locally in your browser with JavaScript. We never see or log the URLs you encode or decode. This matters if you're working with URLs that contain sensitive tokens, search terms, or personal data, you can use this tool with confidence.
